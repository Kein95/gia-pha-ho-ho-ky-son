# Phase 2: Schema & Drizzle Migrations

## Context Links

- [Plan overview](plan.md)
- Reference schema: `docs/schema.sql`
- Reference migration: `docs/migrations/20260312015736_add_lunar_death_date.sql`

## Overview

Convert toàn bộ schema SQL hiện tại sang Drizzle TypeScript schema, generate migrations, push lên Vercel Postgres trống. KHÔNG migrate data ở phase này (Phase 5 lo).

## Key Insights

- **Auth.js v5 schema**: Cần `users`, `accounts`, `sessions`, `verificationTokens` tables (Auth.js Drizzle adapter standard)
- **Profiles table cũ** map sang `users.role` + `users.is_active` (gộp vào users để giảm join)
- **5 RPC functions** sẽ thành TypeScript functions (Phase 3-4), KHÔNG migrate sang DB
- **3 triggers** cần thay:
  - `handle_updated_at` → Drizzle `$onUpdate` hoặc trigger DB (giữ DB trigger đơn giản hơn)
  - `handle_new_user` → BỎ (admin invite-only, không auto-create profile)
  - `handle_first_user_confirmation` → BỎ
- **Enums**: gender_enum, relationship_type_enum, user_role_enum giữ nguyên (Drizzle support pgEnum)
- **15 RLS policies** → BỎ HẾT khỏi DB, app-level enforce (Phase 6)

## Requirements

**Functional:**
- Drizzle schema mirror đúng SQL hiện tại (trừ phần auth/profiles gộp vào users)
- Generate migrations
- Push lên Vercel Postgres trống
- DB connection wrapper

**Non-functional:**
- Schema file split theo bảng (mỗi file <100 lines)
- Type-safe queries trong TypeScript

### 3. Create lib/db/schema/users.ts

Auth.js v5 standard tables + role/is_active gộp vào users:

```ts
import {
  pgTable, text, timestamp, primaryKey, integer, boolean,
} from "drizzle-orm/pg-core";
import { userRoleEnum } from "./enums";
import type { AdapterAccount } from "next-auth/adapters";

export const users = pgTable("users", {
  id: text("id").primaryKey().$defaultFn(() => crypto.randomUUID()),
  email: text("email").unique().notNull(),
  emailVerified: timestamp("emailVerified", { mode: "date" }),
  name: text("name"),
  image: text("image"),
  passwordHash: text("password_hash"),  // bcrypt hash for Credentials provider
  role: userRoleEnum("role").default("member").notNull(),
  isActive: boolean("is_active").default(false).notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
  updatedAt: timestamp("updated_at").defaultNow().notNull().$onUpdate(() => new Date()),
});

export const accounts = pgTable("accounts", {
  userId: text("userId").notNull().references(() => users.id, { onDelete: "cascade" }),
  type: text("type").$type<AdapterAccount["type"]>().notNull(),
  provider: text("provider").notNull(),
  providerAccountId: text("providerAccountId").notNull(),
  refresh_token: text("refresh_token"),
  access_token: text("access_token"),
  expires_at: integer("expires_at"),
  token_type: text("token_type"),
  scope: text("scope"),
  id_token: text("id_token"),
  session_state: text("session_state"),
}, (account) => ({
  pk: primaryKey({ columns: [account.provider, account.providerAccountId] }),
}));

export const sessions = pgTable("sessions", {
  sessionToken: text("sessionToken").primaryKey(),
  userId: text("userId").notNull().references(() => users.id, { onDelete: "cascade" }),
  expires: timestamp("expires", { mode: "date" }).notNull(),
});

export const verificationTokens = pgTable("verificationTokens", {
  identifier: text("identifier").notNull(),
  token: text("token").notNull(),
  expires: timestamp("expires", { mode: "date" }).notNull(),
}, (vt) => ({
  pk: primaryKey({ columns: [vt.identifier, vt.token] }),
}));
```

### 4. Create lib/db/schema/persons.ts

```ts
import { pgTable, uuid, text, integer, boolean, timestamp, index } from "drizzle-orm/pg-core";

export const persons = pgTable("persons", {
  id: uuid("id").defaultRandom().primaryKey(),
  fullName: text("full_name").notNull(),
  gender: genderEnum("gender").notNull(),
  birthYear: integer("birth_year"),
  birthMonth: integer("birth_month"),
  birthDay: integer("birth_day"),
  deathYear: integer("death_year"),
  deathMonth: integer("death_month"),
  deathDay: integer("death_day"),
  deathLunarYear: integer("death_lunar_year"),
  deathLunarMonth: integer("death_lunar_month"),
  deathLunarDay: integer("death_lunar_day"),
  isDeceased: boolean("is_deceased").default(false).notNull(),
  isInLaw: boolean("is_in_law").default(false).notNull(),
  birthOrder: integer("birth_order"),
  generation: integer("generation"),
  otherNames: text("other_names"),
  avatarUrl: text("avatar_url"),
  note: text("note"),
  createdAt: timestamp("created_at").defaultNow().notNull(),
  updatedAt: timestamp("updated_at").defaultNow().notNull().$onUpdate(() => new Date()),

  generationIdx: index("idx_persons_generation").on(t.generation),

  currentResidence: text("current_residence"),
```

### 5. Create lib/db/schema/relationships.ts

```ts
import { pgTable, uuid, text, timestamp, unique, index, check } from "drizzle-orm/pg-core";
import { sql } from "drizzle-orm";
import { relationshipTypeEnum } from "./enums";
import { persons } from "./persons";

export const relationships = pgTable("relationships", {
  id: uuid("id").defaultRandom().primaryKey(),
  type: relationshipTypeEnum("type").notNull(),
  personA: uuid("person_a").notNull().references(() => persons.id, { onDelete: "cascade" }),
  personB: uuid("person_b").notNull().references(() => persons.id, { onDelete: "cascade" }),
  note: text("note"),
  createdAt: timestamp("created_at").defaultNow().notNull(),
  updatedAt: timestamp("updated_at").defaultNow().notNull().$onUpdate(() => new Date()),
}, (t) => ({
  noSelfRel: check("no_self_relationship", sql`${t.personA} != ${t.personB}`),
  uniquePair: unique().on(t.personA, t.personB, t.type),
  personAIdx: index("idx_relationships_person_a").on(t.personA),
  personBIdx: index("idx_relationships_person_b").on(t.personB),
  typeIdx: index("idx_relationships_type").on(t.type),
}));
```

```ts
export const customEvents = pgTable("custom_events", {

  content: text("content"),
  eventDate: date("event_date").notNull(),
  location: text("location"),

}, (t) => ({
  dateIdx: index("idx_custom_events_date").on(t.eventDate),
  createdByIdx: index("idx_custom_events_created_by").on(t.createdBy),
}));
```

## Risk Assessment

| Risk | Mitigation |
|---|---|
| Schema drift giữa Drizzle và SQL gốc | Compare cẩn thận từng cột, dùng `db:studio` verify |
| Enum case sensitivity | Drizzle pgEnum giữ exact case, OK |
| `unique()` constraint conflict với data sau migrate | Phase 5 dedupe data trước khi import |

## Security Considerations
